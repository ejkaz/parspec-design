Color

Two-tier token model: primitive ramps → role tokens. Every CSS variable that consumes color references a role, not a primitive directly. Polaris/Carbon precedent. Surfaces — mode per surface

Brand Artifacts

mode: dark · background: {neutral.950} → #0F0F0F
Examples: decks, marketing, hero-sections, social-proof, conference-banners, investor-decks, sales-pdfs, case-studies

The CEO proposal, sales decks, marketing site hero, and trade-show booth all sit in the dark register. This is the buyer's first impression of Parspec brand at the deal-side. Category leaders (Procore, Bluebeam, Autodesk Construction Cloud, ServiceTitan, Fieldwire) all run dark hero + dark brand artifacts.

Product Ui

mode: light · background: {neutral.200} → #F2F2F2
Examples: app-screens, dashboards, forms, quoting-workspace, submittal-review, pm-tools

The Parspec product UI is light mode — appropriate for data-dense, all-day procurement workflows. Same as Procore, Bluebeam, Autodesk ACC, ServiceTitan, Fieldwire. Brand and product share the same primary axis; only the mode bindings differ.

Documents

mode: light · background: {neutral.0} → #FFFFFF
Examples: contracts, term-sheets, formal-memos, customer-pdfs, invoices, AP-bound-artifacts

Transactional documents (invoices, contracts) stay pure white because that's what accounts-payable systems expect. Per R3 ICP Buyer: "Don't put paper-warm on the invoice itself — the actual transactional document should stay white because that's what AP systems expect." Marketing PDFs and case studies use brand_artifacts mode (dark) instead.
